Overview

evictions.help is an AI-powered self-help document preparation service for tenants facing eviction. Users answer eligibility questions, pay a $395 flat fee, chat with an AI intake specialist, and receive a complete ready-to-file court paperwork packet — including the official state court answer form, fee waiver, legal motions, checklists, hearing preparation guide, and rental assistance directory.

Supported states: AR, AZ, CA, CO, CT, FL, GA, IL, LA, MA, MI, MN, NM, NV, OR, RI, SC, TN, TX, VA (20 states)

User Flow

1. Landing Page → 2. Eligibility (7 questions) → 3. Address + Email → 4. Payment ($395) → 5. AI Chatbot Intake → 6. Summary + Download

Step 1: Landing Page (/)

Serves app/static/index.html. Hero section with pricing, what's included, how it works. Below the fold, 7 eligibility questions (all yes/no except state dropdown):

  1. State (20-state dropdown)
  2. Are you the tenant? (yes/no)
  3. Have you been served? (yes/no — soft warning if no)
  4. Residential rental? (yes/no)
  5. Section 8 / public housing? (yes/no — hard block)
  6. Active military? (yes/no — hard block)
  7. Bankruptcy? (yes/no — hard block)

If all pass → address fields appear (street, city, county, zip, email).

Step 2: Payment

Authorize.net Accept.js hosted payment form. Card data goes directly to Authorize.net (PCI SAQ-A compliant). Backend processes via app/services/payment.py. Sandbox credentials in .env. After payment → redirects to /chat?state=TX&email=...&address=...&county=...

Step 3: AI Chatbot Intake (/chat)

The chatbot uses DeepSeek (deepseek-chat) via OpenAI-compatible API. 7 conversational phases collect all data:

  1. Personal & Location — name, county, address, phone, email
  2. Landlord & Case Info — landlord name, case number, court, service date, amount claimed
  3. Risk Screening — writ of possession, Section 8, military, bankruptcy (hard stops)
  4. Rent & Payment — monthly rent, disputes, repair notices, rental assistance
  5. Defenses — 12 standard defenses with explanations
  6. Preferences — judge/jury, time extension, payment plan
  7. Financial — income, expenses, assets, benefits (for fee waiver)

At completion, outputs structured JSON with all collected data. The system injects state/address/email from URL params if the chatbot didn't collect them.

Step 4: Packet Download

Chatbot shows summary → user clicks "Generate & Download" → POSTs full intake data to /api/v1/documents/generate-packet → receives ZIP with 15-18 PDFs.

Court Form System

Two Filling Approaches

MethodStatesHow It Works
Widget FillCA, CO, CT, FL, GA, IL, MA, MI, NV, RI, SC, TN, TX, VAOriginal PDF has AcroForm fields. Auto-fill detects field names (defendant, plaintiff, case number, etc.) and sets values. Defense checkboxes mapped via defense_options in state config.
Rebuilt + OverlayAR, AZ, MN, NM, OROriginal is scanned/non-fillable. Rebuilt version renders form as background image with standardized widget names. Text overlay for positioning. Defense narrative paragraph lists all defenses.
HybridLABoth widget fill + overlay for complex 14-page form.

State Config Structure

Each state in state_configs.py has this structure:

"TX": {
    "name": "Texas",
    "answer_form": "tx_eviction_answer.pdf",      // Official court form
    "fee_waiver_form": "tx_fee_waiver.pdf",        // Fee waiver form
    "has_fillable_fields": True,                    // Has AcroForm widgets?
    "court_type": "Justice of the Peace Court",
    "field_mapping": {                              // Data key → PDF widget name
        "case_number": "Text1",
        "full_name": "Text5",
        ...
    },
    "defense_options": [                            // Defense → checkbox widget
        {"key": "def_repairs", "field": "Check Box1", "label": "..."},
        ...
    ],
    "overlay_positions": {                          // Coordinate overlay (scanned forms)
        "full_name": {"page": 1, "x": 72, "y": 400, "w": 300, "h": 20, "size": 11},
        ...
    },
    "fee_waiver_mapping": { ... },                  // Fee waiver field mapping
    "fee_waiver_overlay": { ... },                  // Fee waiver overlay positions
}

Deployment

The app is a single Uvicorn process. Deploy on any VPS (DigitalOcean, Hetzner, Railway, Render):

  1. Clone repo: git clone github.com/wmabra/eviction-defense
  2. Set up .env with production keys
  3. Run behind Nginx reverse proxy with Let's Encrypt SSL
  4. Point evictions.help DNS to server
  5. Swap Authorize.net sandbox keys → live in .env
  6. Change Accept.js URL from jstest.authorize.net → accept.authorize.net
