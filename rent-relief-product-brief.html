SaaS Product Brief: Local Rent Relief Assistant

Project Goal: Build a scalable web utility that uses an interactive AI chat interface to pre-screen tenants facing financial hardship, verify their local eligibility for municipal grant programs, collect application data, process a checkout fee via Authorize.Net, and output ready-to-file local grant PDFs.

Core Legal Compliance Rule: This platform is strictly an administrative document preparation and clerical utility. It does not interface with court systems, draft legal defenses, or provide legal advice. It maps user-provided strings onto public municipal financial application forms.

Step 1: AI Chat Intake

  • User interacts with a frontend chat widget linked via API to an enterprise LLM provider (e.g., OpenAI or Anthropic API).
  • The backend must use API developer endpoints with data privacy controls enabled (no data utilized for model training).
  • Data Collected: Zip Code/City/State, Household Size, Gross Household Income, and Confirmation of Financial Hardship (e.g., past due rent statement).
  • The system queries an internal database to confirm if a local program is active and if the user’s income falls below the target Area Median Income (AMI) thresholds (typically 50% or 80%).

Step 2: Paywall (Authorize.Net Integration)

  • If eligible, the UI displays a clean payment terminal.
  • Security Standard: Rent Relief Assistant requires Authorize.Net Accept Suite (Accept.js). Payment forms capture card data securely on the frontend and send it directly to Authorize.Net in exchange for a payment token.
  • The production server never processes, logs, or transmits raw credit card numbers, maintaining clean PCI compliance.
  • A successful payment transaction token webhooks back to trigger the PDF generation workflow.

Step 3: Document Generation Engine

  • The application pulls raw textual answers saved from the chat logs (Names, Address, Overdue amounts, Landlord info).
  • Instead of an AI generating text, a server-side script maps these strict variable strings directly into static fillable fields on matching PDF templates using libraries like pypdf / pdfminer (Python) or pdfform-fill / pdf-lib (Node.js).

Step 4: Secure Asset Delivery

  • The software packages the completed PDFs along with a structured checklist detailing where and how to upload the document to their local government portal.
  • Files are saved on an encrypted storage bucket (e.g., AWS S3) with short-lived presigned URLs and delivered securely to the user via email or web interface.

2. Targeted Launch Strategy & Market Segments

To maximize development efficiency, the database and form mapping will focus on two distinct structural playbooks:

Playbook A: High-Value "One-Form" Centralized States

These states utilize a single unified application and template state-wide. They provide maximum population coverage with minimum PDF layout mapping code.

  • Colorado: Centralized state program via the Colorado Department of Local Affairs (CERA framework), managing unified monthly application intakes.
  • New Jersey: Single master portal for the State Rental Assistance Program (SRAP) managed by the NJ Department of Community Affairs.
  • Illinois: Unified state intake platform operated under the Illinois Housing Help framework (IHDA).
  • Minnesota: Unified "Bring It Home" Rental Assistance model run centrally by Minnesota Housing.

Playbook B: High-Eviction Metro Routing (Fractured States Like Texas)

In highly populated states without a single form, the system will route users using geographical zip code validation, targeting localized cities with disproportionately high eviction filing rates per capita.

Texas High-Volume Blueprint: By writing only four specific document packages, the software captures over 75% of the Texas eviction market.
  • Houston (Harris County): The highest volume market in Texas (77k+ annual filings, ~9% renter filing rate). Maps onto the Harris County Housing Authority and local municipal utility templates.
  • Dallas & Fort Worth (Dallas/Tarrant Counties): Massive combined metro region. Focus on City of Dallas housing stability formats and Tarrant County community action programs.
  • San Antonio (Bexar County): High per-capita filing area (~27k annual filings, affecting 9 out of 100 renters). Maps onto City of San Antonio Risk Mitigation portals.
  • Austin (Travis County): Highest relative increase in text tracking metrics (~30% year-over-year surge). Maps onto Travis County Health & Human Services Emergency formats.
